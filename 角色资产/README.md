# 红鸾笑 · 角色资产目录规范

> 角色资产是角色参考图的"主版本库"。所有角色定调、跨姿态参考、回填依据都从本目录出发。

## 目录结构

```
角色资产/
├── README.md                    本文件：目录规范与流程
├── 陆雪琪/                      主角色之一
│   ├── 角色设定.md              角色档案 + 生图提示词 + 回填指引
│   ├── 参考图/                  待生成 / 生成中的工作副本（.png / .jpg）
│   └── 归档/                    确认后的归档副本（命名固定，禁止修改）
├── 张小凡/
├── 大黄/
└── 小灰/
```

## 命名规范

### 角色子目录
每个角色固定两个子目录：
- **参考图/**：生图工作副本。生成后未确认的图放这里，命名 `work_<角色>_<视角>_<序号>.png`。
- **归档/**：确认定调后的副本。命名 `archive_<角色>_<视角>_v<版本号>.png`。

参考图确认后才拷贝到归档，文件名固定下来，作为后续所有镜头合成 / 改图 / 视频生成的"母版"。

### 跨目录回填

第1集 HTML 制作脚本中，参考图通过相对路径引用：

```html
<figure class="asset-figure">
  <img src="../../角色资产/陆雪琪/归档/archive_陆雪琪_3view_v1.png" alt="...">
  <figcaption>红鸾笑/角色资产/陆雪琪/归档/archive_陆雪琪_3view_v1.png</figcaption>
</figure>
```

> 制作脚本目录下不再单独存放角色资产副本（场景资产仍保留 `assets/` 子目录作为副本，因为场景母场景需要反复在 HTML 中显示）。角色资产直接通过相对路径回到主目录，符合"资产路径的唯一可信来源"原则。

## 操作流程

```
[1] 在 角色设定.md 中写好生图提示词
   ↓
[2] 调用生图工具（Midjourney / SD / LoRA 等）生成多张候选
   ↓
[3] 候选图先放入 参考图/ 目录
   ↓
[4] 用户确认某张为定调 → 把该图复制到 归档/，文件名固定
   ↓
[5] 把归档路径回填到 角色设定.md 与 HTML 制作脚本对应镜头
```

## 与第1集 HTML 的衔接

HTML 中每个引用角色的镜头（如镜 1、2、3、9、21）都需要：

1. 在该镜的 `参考图备注` 区插入归档图（用相对路径）
2. 在该镜的 `参考图生成提示词` 区指向 `角色设定.md` 中的具体生图提示词
3. 在该镜的 `未来视频素材` 与 `视频生成提示词` 区暂留待生成位置

只有三者同时完成，该镜才算真正"补齐"。

## 跨集复用约定

- **陆雪琪**：3 集主力，需要多版本（白衣日常、道袍首座、眼神细微差）
- **张小凡**：第1集粗衣麻裤，第2集起道袍，**两套造型要明确分离**，不要混用参考
- **大黄 / 小灰**：贯穿全集，造型几乎不变，但镜头环境差异大

参考图归档时，建议使用子版本号区分（如 `v1` 第1集版、`v2` 第2集版），避免造型漂移。
